# create-gnome-launcher

Bash-скрипт для GNOME/Nautilus, который добавляет пункт в меню `Сценарии` и позволяет создать ярлык приложения (`.desktop`) для выбранного исполняемого файла или AppImage.

## Что делает скрипт

- Устанавливает себя в `~/.local/share/nautilus/scripts`.
- Добавляет пункт `Сценарии -> Создать ярлык в меню запуска`.
- Создает `.desktop` файл в `~/.local/share/applications`.
- Предлагает выбрать иконку через диалог.
- Если выбрана `.jpg/.jpeg` и доступен ImageMagick (`magick` или `convert`), конвертирует иконку в `.png`.

## Требования

- GNOME с файловым менеджером Nautilus.
- `bash`
- `zenity`
- `update-desktop-database` (обычно пакет `desktop-file-utils`)

Опционально:

- ImageMagick (`magick` или `convert`) для конвертации JPG/JPEG иконок в PNG.

## Установка

Из папки проекта:

```bash
chmod +x create-gnome-launcher.sh
./create-gnome-launcher.sh --install
```

После установки перезапустите Nautilus:

```bash
nautilus -q
```

## Использование

1. В Nautilus нажмите правой кнопкой на исполняемый файл или AppImage.
2. Выберите `Сценарии -> Создать ярлык в меню запуска`.
3. Введите имя ярлыка.
4. При необходимости выберите иконку.

После этого ярлык появится в меню приложений GNOME.

## Примечания

- Скрипт обрабатывает только первый файл из выделения.
- Если иконка не выбрана, используется `application-x-executable` или изображение рядом с файлом (`.png`, `.jpg`, `.jpeg` с тем же именем).
- Идентификатор ярлыка формируется из имени в безопасный формат (нижний регистр, допустимые символы).
